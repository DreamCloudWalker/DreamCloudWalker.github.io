## 天空盒（Skybox）

天空盒是用一张**立方体贴图（Cube Map）**把整个场景"包"起来，营造无限远的天空、远景的效果。本 Demo 已经在背景渲染了天空盒，取消勾选左侧的「**隐藏边界**」，可以把 Cube Map 6 个面的接缝用黄色线条画出来，帮助你看清这个"包裹着相机的立方体"到底长什么样。

### 一、Cube Map 是什么

普通 2D 纹理用 `(u, v)` 两个坐标采样。Cube Map 则是 **6 张 2D 纹理**拼成的一个立方体，分别对应 +X / -X / +Y / -Y / +Z / -Z 六个面：

```
posX.png  负责 +X 面（右）
negX.png  负责 -X 面（左）
posY.png  负责 +Y 面（上）
negY.png  负责 -Y 面（下）
posZ.png  负责 +Z 面（前）
negZ.png  负责 -Z 面（后）
```

采样时用的是一个**三维方向向量** `(x, y, z)`，而不是 2D 的 uv。GPU 根据这个向量指向哪个面、打在面上的哪个点，取出对应颜色：

```glsl
uniform samplerCube uSkybox;
gl_FragColor = textureCube(uSkybox, dir);  // dir 是方向向量
```

你可以把它想象成：站在立方体正中心，朝某个方向看出去，看到的就是那个方向上墙面的图案。**取消「隐藏边界」**后看到的黄色线条，就是这 6 面墙两两相接的 12 条棱。

### 二、天空盒为什么"无限远"

天空盒的关键技巧有两点：

**1. 立方体始终以相机为中心。** 渲染天空盒时，把视图矩阵的**平移部分去掉**，只保留旋转：

```js
viewMatrix[12] = viewMatrix[13] = viewMatrix[14] = 0;
```

这样不管相机走到哪里，天空盒都跟着相机平移，你永远走不到天空盒的"墙边"——天空看起来就在无限远处。但相机**旋转**时天空盒不动，所以转头能看到天空的不同部分。本 Demo 的边界线用了同样的"去平移"处理，所以无论怎么拖动旋转，黄色立方体始终套在你周围。

**2. 深度永远在最远处。** 在顶点着色器里把天空盒的深度强制设为最远：

```glsl
gl_Position = pos.xyww;   // 令 z = w，透视除法后 z/w = 1，即最远平面
```

这样任何实际物体都会画在天空盒前面，天空盒只填充没有物体遮挡的空白区域。本工程的天空盒 shader 里写的是 `gl_Position.z = 1.0`，边界线 shader 里写的是 `gl_Position.z = pos.w`，都是同一个目的。

### 三、实现要点

天空盒通常**不需要画一个真正的立方体**。本工程的做法是画一个**铺满屏幕的四边形**，在片元着色器里用「逆视图投影矩阵」把每个像素反算回一条世界空间的方向向量，再用它采样 Cube Map。这样只需 2 个三角形就能渲染整个天空，效率很高。

而本 Demo 的**边界线**是另一套简单着色器：直接把一个单位立方体的 12 条棱（24 个顶点）用 `gl.LINES` 画出来，套用"去平移的视图矩阵 + 投影矩阵"，于是这些棱正好和天空盒的 6 个面接缝重合。

### 小结

- Cube Map = 6 张 2D 纹理组成的立方体，用**方向向量**采样（`textureCube`）；
- 天空盒通过**去掉视图平移**实现"无限远"，通过**强制最远深度**保证不遮挡任何物体；
- 实际渲染常用"全屏四边形 + 逆 VP 矩阵反算方向"的技巧，而非真画立方体；
- 取消「隐藏边界」可直观看到这个包裹相机的立方体的 12 条棱。
