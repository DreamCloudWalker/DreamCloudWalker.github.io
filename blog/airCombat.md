## 空战模拟（Air Combat）

这是一个把前面多个模块拼起来的小综合作品：**天空盒 + 飞机模型 + 无限地形 + 序列帧爆炸**。飞机始终向前飞，你用键盘控制它的姿态在地形上空飞行。

### 操作

| 键 | 动作 |
|---|---|
| **A / D** | 向左 / 向右**桶滚**（roll） |
| **W / S** | **俯仰**（pitch，抬头/低头） |
| **Q / E** | **偏航**（yaw，转速较慢） |

飞机**持续向前飞**；撞到地形会播放**爆炸动画**并提示坠毁，1 秒后在高空重生。右窗口是第三人称上帝视角，飞机始终在画面中央。

### 一、用四元数表示飞机姿态

自由飞行的飞机会绕三个机体轴任意旋转，如果用**欧拉角**（绕固定的 X/Y/Z 轴分别转）会遇到**万向节锁（gimbal lock）**和"旋转顺序"的麻烦。所以这里用**四元数**表示朝向：

- 维护一个朝向四元数 `q`；
- 每帧按当前机体轴算出**世界空间**的前向 / 上向 / 右向；
- 把这一帧的角速度做成一个增量四元数 `dq = quat(轴, 角速度·dt)`，**左乘**到 `q` 上：`q = dq * q`。

绕"当前机体轴"增量旋转，正是飞行器操控的自然方式——桶滚就是绕机头前向轴转，俯仰是绕翼展轴转，偏航是绕机身上轴转。

### 二、姿态怎么变成模型矩阵

引擎里飞机的渲染走的是既有的 `drawObject`，它用 `mRotAxis / mRotAngle`（轴角）来旋转模型。所以每帧把四元数转成轴角写进去即可，无需改渲染代码：

```
q（四元数） → (axis, angle)（轴角）→ drawObject 里 mat4.rotate(model, angle, axis)
```

前向位移则直接 `pos += forward * speed * dt`，写进模型的平移。

### 三、第三人称跟随相机

相机机位放在飞机的**后上方**，朝向飞机前方一点：

```
eye    = planePos - forward * 后退距离 + up * 抬高
center = planePos + forward * 前视距离
viewMatrix = lookAt(eye, center, up)
```

`forward / up` 都由飞机当前四元数旋转机体局部轴得到，所以相机会随飞机翻滚俯仰一起动，始终"挂"在飞机背后。右窗口上帝视角用同样的方式，只是退得更远、抬得更高，让飞机居中可见。

为了能看到远处地平线，本 Demo 把**投影矩阵的远裁剪面拉到 1000**（主视口和上帝视角都改），并同步更新上帝视角的视锥体。切换到别的 Demo 时，飞机状态和投影矩阵都会复位回原值。

### 四、撞地检测与爆炸重生

每帧用飞机的世界位置去查**地形高度场** `terrainHeightAt(x, z)`：当飞机高度低于地面一定阈值就判定坠毁。坠毁时：

1. 把飞机藏起来；
2. 在坠毁位置画一个**正对相机的 billboard**，播放《精灵图集》Demo 里那张 explosion.png 的序列帧爆炸；
3. 提示"飞机已坠毁"；
4. 计时 1 秒后复位飞机到高空，重新开始。

爆炸 billboard 用相机的右/上轴张成四边形，所以无论从主视口还是上帝视角看，它都正对着你。

### 小结

- 飞行姿态用**四元数**累积、绕机体轴增量旋转，避免万向节锁；
- 朝向转成**轴角**喂给既有 `drawObject`，位移直接写平移，无需改渲染；
- 第三人称相机挂在飞机后上方，远裁剪面拉到 1000 看远景；
- 撞地用**地形高度场**检测，坠毁播放序列帧爆炸 billboard，1 秒后重生；
- 这个作品复用了天空盒、PBR 飞机、无限 LOD 地形、精灵图集四个模块。
