## 混合与透明排序

这个 Demo 用 5 个**斜向交错排列、互不相交、深度各不相同**的半透明球，来演示半透明物体渲染中最容易踩坑的三个点。左侧三个开关分别对应一个 OpenGL 概念，建议你逐个打开/关闭，观察画面变化。

### 一、什么是混合（Blending）

不透明物体渲染时，新像素直接覆盖旧像素。但半透明物体需要让背后的颜色"透"过来，这就要靠**混合**。

WebGL 中开启混合：

```js
gl.enable(gl.BLEND);
gl.blendFunc(gl.SRC_ALPHA, gl.ONE_MINUS_SRC_ALPHA);
```

混合方程为：

```
最终颜色 = 源颜色 × src.a + 目标颜色 × (1 - src.a)
```

其中"源颜色"是当前正在画的片元，"目标颜色"是帧缓冲里已有的颜色，`src.a` 是当前片元的 alpha（不透明度）。

> 试试看：**关闭"开启混合"**，5 个球就变成完全不透明，互相直接遮挡，看不出任何透明效果。

### 二、为什么要关闭深度写入（depthMask）

深度测试（depth test）保证近处物体遮挡远处物体。但深度测试有两个动作：

1. **深度比较**：当前片元比已有的近吗？近就通过。
2. **深度写入**：通过后，把当前片元的深度写进深度缓冲。

问题在于：半透明物体**不应该挡住它背后的东西**，但只要它写入了深度，后面再画的、位于它身后的物体就会被深度测试丢弃，导致背后的球"凭空消失"，出现空洞或穿帮。

解决办法：渲染半透明物体时，**保留深度比较，但关闭深度写入**：

```js
gl.depthMask(false);   // 只读深度，不写深度
// ... 画完所有半透明物体后
gl.depthMask(true);    // 恢复
```

> 试试看：开启混合后，**保持勾选"写入深度"并关闭排序**，你会看到某些球被错误地"切掉"一块——这就是半透明物体写深度造成的穿帮。取消勾选"写入深度"，穿帮消失。

### 三、为什么要由远到近排序

即使关闭了深度写入，**画的先后顺序仍然影响最终颜色**。因为混合方程不满足交换律：先画红再画蓝，和先画蓝再画红，叠出来的颜色不一样。

正确的做法是：把所有半透明物体按到相机的距离**从远到近**排序，先画最远的，再依次画近的。这样每一次混合时，"目标颜色"都已经是它背后所有物体正确叠加后的结果。

本 Demo 的排序逻辑（在相机空间按 z 排序）：

```js
// 把球心变换到相机空间，-z 越大表示离相机越远
order.sort((a, b) => b.dist - a.dist);  // 远的排前面，先画
```

> 试试看：**开启混合 + 关闭深度写入**后，再切换"由远到近排序"开关。不排序时颜色叠加是乱的，排序后过渡才正确自然。

### 小结

半透明物体的正确渲染口诀：

1. **开启混合** —— 才能透出背景；
2. **关闭深度写入**（保留深度测试）—— 避免半透明物体挡掉身后的东西；
3. **由远到近排序** —— 保证混合顺序正确。
