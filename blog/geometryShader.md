## 几何着色器（Geometry Shader）

> ⚠️ **WebGL（1 和 2）都没有几何着色器**，也没有细分着色器——这两个阶段从未纳入 WebGL 规范。本 Demo 用 **CPU 预生成几何**来模拟几何着色器最经典的教学用途：**法线可视化**。

这个 Demo 画了一个球体，并从它表面**伸出一根根短线表示法线方向**（红=顶点法线，黄=面法线）。在真正支持几何着色器的桌面 OpenGL 里，这些线段是几何着色器在 GPU 上**即时生成**的；WebGL 没有该阶段，所以我们在初始化时用 CPU 把线段顶点算好放进缓冲区，再用 `gl.LINES` 画出来——视觉效果和教学含义完全一致。

### 一、几何着色器在管线里的位置

简化的可编程渲染管线：

```
顶点着色器  →  [细分着色器]  →  [几何着色器]  →  光栅化  →  片元着色器
 逐顶点          逐面片          逐图元                      逐像素
```

- **顶点着色器**：逐顶点，输入一个顶点、输出一个顶点。**不能增删顶点**。
- **几何着色器**：逐**图元**（点/线/三角形）运行，最特别的能力是——**可以凭空生成新图元**。输入一个三角形，它可以输出 0 个、1 个、或好几个图元。
- **片元着色器**：逐像素，**不能改变几何**。

所以"给每个顶点画一根法线线"这种**需要无中生有造出新线段**的事，顶点和片元着色器都做不到，正是几何着色器的典型活儿。

### 二、几何着色器的经典用途

1. **法线 / 切线可视化**（本 Demo）：每个顶点或每个面吐出一根线段，调试法线方向是否正确。
2. **点 → 公告板（billboard）**：输入一个点，输出一个始终朝向相机的四边形——粒子系统、草地、雪花常这么做（一个点变成一个面片，省带宽）。
3. **图元放大 / 爆破**：把每个三角形沿法线方向推出去，做"模型炸开"特效。
4. **轮廓线 / 毛发（fur shell）**：沿边或沿法线生成额外几何。

它们的共同点都是：**在 GPU 上动态增加几何**，而不是 CPU 提前算好所有顶点再传上去。

### 三、本 Demo 怎么"模拟"它

法线线段的生成逻辑（在真实管线里属于几何着色器，这里放在 CPU）：

- **顶点法线**：球面每个顶点 `P`、法线 `N`，生成一条线段 `P → P + N·len`。
- **面法线**：每个三角形求质心 `C` 和面法线 `(b-a)×(c-a)`，生成 `C → C + faceN·len`。

```js
// 顶点法线线段（伪代码）
for (每个顶点 P, N) {
    线段.push(P);            // 起点：表面上
    线段.push(P + N * len);  // 终点：沿法线伸出
}
```

把这些线段顶点一次性塞进 VBO，渲染时用 `gl.LINES` 画。拖动旋转球体，能直观看到：

- **顶点法线**（红）：球面每个顶点伸出一根，方向都指向球心外，分布均匀；
- **面法线**（黄）：每个三角形中心一根，能看出网格是由一个个平面三角形拼出来的。

> 切换「顶点法线 / 面法线」、拖动「法线长度」滑块观察。顶点法线常用于平滑着色（相邻面共享、插值得到圆滑感），面法线则代表每个三角形真正的朝向。

### 四、WebGL 里没有它，怎么办

既然 WebGL 没有几何着色器，这些效果在 WebGL 里通常这样替代：

- **法线可视化 / 爆破**：CPU 预生成额外几何（像本 Demo），或用顶点着色器做有限的顶点位移；
- **点 → 公告板**：用**实例化（instancing）**或预先把每个点扩展成 4 个顶点的四边形，在顶点着色器里朝向相机；
- 真要 GPU 动态增删几何，得用 **WebGPU** 的 compute shader，或退回桌面 OpenGL / Vulkan。

### 小结

- 几何着色器是管线里**逐图元**的可选阶段，独特能力是**动态生成新图元**；
- 经典用途：法线可视化、点→公告板、图元爆破、轮廓/毛发；
- **WebGL 没有几何着色器（也没有细分着色器）**，本 Demo 用 CPU 预生成法线线段来模拟其效果；
- 在 WebGL 里替代方案是实例化 / 预扩展几何 / 顶点位移，真正的 GPU 造几何要靠 WebGPU 或桌面图形 API。
