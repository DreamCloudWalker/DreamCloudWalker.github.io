## 环境映射（Environment Mapping）

这个 Demo 在天空盒中央放了一个球体，球面像**镜面金属球**或**玻璃球**一样，映出周围天空盒的景象。它复用了「天空盒」Demo 已经加载好的那张 **CubeMap**（立方体贴图），不重复加载资源。

环境映射的核心思想：用一张包住整个场景的 CubeMap 当作"环境"，物体表面不去做真实的光线追踪，而是**根据表面朝向直接去 CubeMap 上采样**一个方向的颜色，廉价地伪造出反射 / 折射效果。

### 一、反射（Reflection）

光线照到镜面会按法线**反射**。在着色器里，已知：

- `N`：表面法线（世界空间）；
- `V`：视线方向（相机 → 表面）。

反射方向用内置函数 `reflect` 一步算出，再用它采样 CubeMap：

```glsl
vec3 N = normalize(vWorldNormal);
vec3 V = normalize(vWorldPos - uViewPos);   // 相机指向表面
vec3 R = reflect(V, N);                      // 反射方向
vec3 color = textureCube(uEnvMap, R).rgb;    // 用方向向量采样立方体贴图
```

关键点：**采样 CubeMap 用的是一个方向向量**（不是 0~1 的 UV）。GPU 会根据这个方向射线落在立方体 6 个面的哪个位置，取出对应纹素。转动视角或旋转球体时，`R` 随之改变，球面上就"流动"出周围环境的倒影。

### 二、折射（Refraction）

光线穿过玻璃、水这类透明介质时会发生**弯折**，弯折程度由两种介质的折射率之比 `eta` 决定（斯涅尔定律）。同样有内置函数：

```glsl
vec3 R = refract(V, N, uEta);              // 折射方向
vec3 color = textureCube(uEnvMap, R).rgb;
```

`eta = n₁ / n₂`。空气→玻璃约为 `1.0 / 1.5 ≈ 0.66`。`eta` 越小，弯折越强，玻璃球的"放大/扭曲"效果越夸张。

### 三、菲涅尔混合（更真实）

现实里没有纯反射或纯折射的物体：**正对着看**玻璃时主要是透光（折射），**掠射边缘**时几乎全是反射（想想平静水面，脚下清澈、远处反光）。这种"角度决定反射比例"的现象就是**菲涅尔效应**，用 Schlick 近似在反射和折射之间混合：

```glsl
float NoV = max(dot(N, -V), 0.0);           // 视线与法线夹角余弦
float F = 0.04 + 0.96 * pow(1.0 - NoV, 5.0); // 边缘 NoV→0 时 F→1（全反射）
vec3 color = mix(refractCol, reflectCol, F);
```

切到「菲涅尔混合」模式，盯着球心和球的边缘对比看：中心偏透明折射、边缘偏镜面反射，质感比纯反射/纯折射都自然。这与「菲涅尔效果」Demo 里的边缘光是同一套数学。

### 四、和天空盒的关系

天空盒本身也是用方向向量采样同一张 CubeMap（采样的是"视线方向"），所以球面的反射和背景天空盒**天然吻合** —— 球面上看到的，正是它"应该"映出的那片天空。这也是为什么环境映射通常和天空盒搭配使用。

> 局限：环境映射假设环境是"无限远"的（只跟方向有关，与物体位置无关），所以它**反射不出场景里的其它物体**，也没有真实的多次反射/自反射。要那些效果就得用动态 CubeMap、平面反射或光线追踪。但作为低成本的镜面/玻璃近似，它至今仍被大量使用。

### 小结

- 环境映射用一张包裹场景的 **CubeMap** 当环境，按**表面朝向**直接采样，廉价伪造反射/折射；
- 采样 CubeMap 用**方向向量**：反射方向 `reflect(V, N)`、折射方向 `refract(V, N, eta)`；
- **菲涅尔**按视角在反射/折射间混合，边缘偏反射、正面偏折射，最真实；
- 与天空盒共用同一张 CubeMap，反射与背景天然一致；
- 代价是反射不到场景内其它物体（环境被视为无限远）。
