## 分形着色器（Fractal Shader）

这个 Demo 整屏铺了**一块全屏四边形**，画面上看到的每一个像素颜色，都是片元着色器里一段**逃逸时间算法**实时算出来的 —— 全程**没有用到任何贴图**。这是理解"**片元着色器 = 在每个像素上独立运行的一段小程序**"最纯粹的例子。

### 一、逃逸时间算法

以经典的 **Mandelbrot 集**为例。把屏幕上每个像素映射到复平面上的一个点 `c`，然后反复迭代这个公式（`z` 从 0 开始）：

```
z → z² + c
```

观察 `z` 要多少步才会"逃逸"到半径 2 以外（`|z| > 2`）：

- **很快逃逸**：这个点离集合远，按步数上色；
- **一直不逃逸**（迭代到最大次数仍 `|z| ≤ 2`）：这个点属于 Mandelbrot 集本身，涂黑。

把每个像素的逃逸步数映射成颜色，就得到了那张著名的分形图案。整个过程不需要任何顶点几何或纹理，纯粹是**数学在 GPU 上逐像素并行计算**。

### 二、片元着色器里的核心循环

复数平方 `(x + yi)² = (x² − y²) + (2xy)i`，对应到 `vec2`：

```glsl
vec2 c = uCenter + vec2(vNdc.x * uAspect, vNdc.y) * uScale;  // 像素 → 复平面
vec2 z = vec2(0.0);

float iter = 0.0;
const int MAX = 1000;          // GLSL ES 1.0 的循环上界必须是常量
for (int i = 0; i < MAX; i++) {
    if (i >= uMaxIter) break;  // 真正的上限由 uniform 控制
    z = vec2(z.x*z.x - z.y*z.y, 2.0*z.x*z.y) + c;  // z = z² + c
    if (dot(z, z) > 4.0) break;                    // |z|² > 4 即逃逸
    iter += 1.0;
}
```

> **WebGL1 / GLSL ES 1.0 的限制**：`for` 循环的判断条件里**不能直接用 uniform 变量**做上界，必须是编译期常量。常见写法就是给一个足够大的常量上界 `MAX`，循环体内再用 `if (i >= uMaxIter) break;` 提前结束。

### 三、平滑着色

只用整数步数上色会出现一圈圈明显的**色带（banding）**。用"连续逃逸值"可以把台阶抹平：

```glsl
float sn = iter - log2(log2(dot(z, z))) + 4.0;
vec3 col = palette(sn * 0.02);
```

`palette()` 用余弦函数生成平滑渐变的彩虹 / 火焰 / 冰蓝配色。

### 四、Mandelbrot 与 Julia

两者用的是**同一个迭代公式**，区别只在初值：

| | `z` 初值 | `c` |
|---|---|---|
| **Mandelbrot** | `0` | 像素坐标 |
| **Julia** | 像素坐标 | 固定常数（可调） |

所以勾选「Julia 集」后，再拖动「Julia 常数 c」的 `cx / cy` 滑块，就能看到分形图案**连续地变形**，非常直观地展示了这个常数对形状的控制。

### 五、交互

所有交互都是**改 uniform**，shader 本身不变：

- **缩放**：改 `uScale`（复平面半高），越小放大倍数越高 —— 分形的"无限自相似细节"就是这么放出来的；
- **最大迭代次数**：放得越大，越需要更多迭代才能分辨边界细节，否则深处会糊成一团（但迭代越多越耗 GPU）；
- **配色 / 模式**：切换调色板和 Mandelbrot/Julia。

> 注意：用了 `precision highp float`。如果用 `mediump`，放大到一定倍数后就会因精度不足出现色块、抖动 —— 这也顺带说明了**着色器精度限定符**的实际影响。受限于 32 位浮点精度，放大倍数也有上限，再往里放大会"像素化"。

### 小结

- 片元着色器是**逐像素并行**运行的小程序，分形是它最直观的算力展示；
- 逃逸时间算法：迭代 `z = z² + c`，按逃逸快慢上色，永不逃逸者即集合本身；
- GLSL ES 1.0 循环上界须为常量，用 `break` 控制真实迭代次数；
- Mandelbrot 与 Julia 同式不同初值；交互全靠改 uniform；
- `highp` 精度与 32 位浮点上限决定了能放大的极限。
