## 视差贴图（Parallax Mapping）

这个 Demo 并排展示三堵砖墙，逐步叠加贴图技术，展示从"纯平面"到"以假乱真"的演变：

- **左墙**：仅漫反射贴图 —— 砖缝是画上去的，完全平坦
- **中墙**：漫反射 + 法线贴图 —— 砖缝有法线扰动产生的明暗凹凸，但**掠射角下会穿帮**
- **右墙**：漫反射 + 法线贴图 + 视差贴图 —— 砖缝被真正"推了进去"，掠射角下也有真实的深度遮挡

### 一、法线贴图的缺陷

法线贴图通过扰动法线方向伪造凹凸，但它**不会改变纹理采样位置**——不管法线被弯折到哪里，你取的始终是同一个 UV 坐标处的颜色。

问题在掠射角（视线几乎平行于表面）时暴露：
- 你应该看到砖的侧面遮挡住砖缝——但法线贴图没有改变纹理坐标，砖缝的颜色照样被画出来
- 砖缝再暗、法线再弯折，它仍然是**贴在平面上的**——侧面看没有深度感

视差贴图解决的就是这个：**根据视线角度和高度信息，偏移纹理采样坐标**，让高处（砖面）的纹素"浮出来"，低处（砖缝）的纹素"推进去"。

### 二、视差贴图的原理

视差贴图需要一张**高度图（Height Map / Displacement Map）**：灰阶图片，亮=高（砖表面），暗=低（砖缝）。

核心公式（在片段着色器中逐像素执行）：

```glsl
// 1. 把世界空间视线方向转到切线空间
vec3 V_world   = normalize(uViewPos - vWorldPos);
vec3 V_tangent = vec3(
    dot(V_world, vTBN[0]),   // 世界→切线：TBN 矩阵的逆 = 转置
    dot(V_world, vTBN[1]),
    dot(V_world, vTBN[2])
);

// 2. 采样高度图
float height = texture2D(uHeightSampler, vTexCoord).r;

// 3. Offset Limiting：除以 z 避免掠射角偏移过大
vec2 offset = V_tangent.xy / (abs(V_tangent.z) + 0.001) * (height * scale - bias);

// 4. 用偏移后的 UV 采样漫反射和法线贴图
vec2 parallaxUV = vTexCoord + offset;
vec4 diffColor  = texture2D(uDiffuseSampler, parallaxUV);
vec3 tn         = texture2D(uNormalSampler, parallaxUV).rgb;
```

### 三、为什么除以 V_tangent.z（Offset Limiting）

视线越贴近表面（掠射角），`V_tangent.xy` 越大，`V_tangent.z` 越小。如果直接按 xy 偏移，掠射角下偏移量会巨大，纹理会"游泳"甚至撕裂。

除以 `V_tangent.z` 是一种**偏移限制（Offset Limiting）**技巧——它利用 `xy/z` 的性质：当视线接近平行表面时，z 接近 0，xy 也相应增大，比值保持在可控范围。这对掠射角下的画面稳定至关重要。

### 四、对比：三种贴图技术

| 左墙 | 中墙 | 右墙 |
|---|---|---|
| 贴图样本不变 | 法线扰动 | UV 被视差偏移 |
| 无凹凸，纯"墙纸" | 有明暗"凹凸" | 真正的深度偏移 |
| 掠射角：完全平坦 | 掠射角：穿帮（砖缝仍在原位） | 掠射角：砖缝被推进去 |

> **试试看**：把右边墙面旋转到几乎平行视线的角度（接近掠射角）。对比中墙和右墙——中墙的法线凹凸此时和平面无异（砖缝被压缩到几乎没有），右墙的视差偏移依然让砖缝"沉下去"。这就是法线贴图和视差贴图在掠射角下的本质区别。

> 拖动「偏移强度」滑块：太小 → 和法线贴图差不多；太大 → 纹理撕裂。合适的值大约 0.04–0.08，取决于高度图的对比度。

### 五、高度图的约定

`bricks_parallax.jpg` 是一张灰阶图：
- 白色(≈1.0) = 砖面（高）
- 深色(≈0.2) = 砖缝（低）

`height * scale - bias` 这条公式里，`bias` 的作用是让平坦区域（高度接近 bias）不发生偏移，只有砖缝区域（高度低于 bias）才被"推进去"。这也是一个质量控制参数。

### 六、局限性

视差贴图有几种变体，本 Demo 实现的是**基础视差贴图（Parallax Mapping with Offset Limiting）**：

- **过于陡峭的角度**（几乎完全侧看）仍会纹理拉伸和撕裂——这时需要更高级的技术
- **陡峭视差贴图（Steep Parallax Mapping）**：沿视线方向分层采样（ray marching），质量更好但更耗性能
- **视差遮蔽贴图（Parallax Occlusion Mapping, POM）**：在陡峭视差的基础上做层间插值 + 自阴影，效果最接近真正的位移，但计算量也最大——常用于现代 3A 游戏中的地面和墙壁细节

### 小结

- 法线贴图扰动法线但不动 UV → 掠射角穿帮；
- 视差贴图**根据高度和视线偏移 UV** → 掠射角下砖缝真正被遮挡；
- 偏移需要除以 `V_tangent.z` 做**偏移限制**，防止拉伸和撕裂；
- height × scale − bias 控制偏移幅度和偏置，需根据高度图调整；
- 更高级的 POM 可达到接近几何位移的质量，但本 Demo 的带偏移限制的视差贴图是理解的基础。
